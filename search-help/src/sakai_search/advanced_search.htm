Advanced Search

The Search Tool is driven by an index that contains multipl sub indexes. Whne using the tool for basic search only 2 of these indicies are used. The context index, and the content index. The content index contains a stored and tokenized full text index of a digested form of the content. This means that a Word document, uploaded to resources will have had all its text extracted and indexed, whereas a video package may have had only the metadata indexed. When the search results are returned, it is this content that is highlighted with the matching terms.

The context index provides and keyword index of the source of the content. This is the worksite or course id which contains the content. When the search tool performes a basic search it prepends a search term of +content:<currentsiteid< where <currentsiteid> is the context or siteid where the search is being performed. This can be translated as results must come from the current site.

If performing and advanced search you can search for terms within any of the indexes. For example to search only in announcements you can use the search terms +tool:announcement +content:cowslip which will only return announcements containing the word cowlisp.

The index currently contains the following named indexes.

  • content: A Tokenized, stored index of the digested content of the search documents
  • context: A Keyword stored index of the source context of the search document.
  • tool: A Keyword stored index of the tool name producing the search document.
  • title: A Tokenized stored index of the title of the search document.
  • reference: A Keyword stored index of the Sakai Entity reference.
